
Model
대학교 사내 데이터 시각화 대시보드 개발 프로젝트 기술 스택 추천
[프로젝트 개요]
대학교 내부 데이터를 직관적으로 파악하고 공유하기 위한 웹 기반 시각화 대시보드 구축. 이카운트 시스템에서 추출한 엑셀 데이터를 기반으로 주요 성과 지표(실적, 논문, 학생 수, 예산 등)를 시각화하여 제공하며, 내부 직원들이 로그인하여 편리하게 이용할 수 있도록 함.

[판단 기준]
AI 구현에 적합하며 인기 있는 기술
믿을 만한 기업에 의해 활발히 유지보수
Breaking Change가 잦지 않고 하위 호환성 보장
기본 요건: Django Rest Framework 사용, Railway 배포

[기술 스택 추천]
1. 백엔드 (Backend)
메인 프레임워크: Django & Django Rest Framework (DRF)
선정 이유: 프로젝트의 기본 요건입니다. 강력한 ORM, 관리자 기능, RESTful API 구축에 최적화된 DRF는 데이터 모델링 및 관리에 용이하며, 뛰어난 보안 기능과 안정성을 제공합니다. Google, Instagram 등에서 사용하여 신뢰도가 높고, 활발한 커뮤니티와 유지보수가 이루어져 하위 호환성도 잘 보장됩니다.
데이터베이스: PostgreSQL
선정 이유: 관계형 데이터베이스 중 가장 강력하고 안정적이며, 대량의 데이터 처리와 복잡한 쿼리에 강합니다. 오픈 소스이지만 기업 환경에서 널리 사용되며 활발히 유지보수됩니다. 다양한 데이터 타입을 지원하며 확장성이 뛰어납니다. Django와의 연동도 쉽고 안정적입니다.
데이터 파싱 및 처리: Pandas (Python 라이브러리)
선정 이유: AI 및 데이터 분석 분야에서 가장 널리 사용되는 Python 라이브러리입니다. 엑셀 파일(CSV, XLSX 등)을 읽고 쓰고, 데이터를 가공, 분석, 변환하는 데 최적화되어 있습니다. 직관적인 API를 제공하여 개발 효율성이 높으며, 활발한 커뮤니티와 꾸준한 업데이트가 이루어집니다.
배포: Railway
선정 이유: 프로젝트의 기본 요건입니다. Django 애플리케이션 배포에 적합하며, CI/CD 통합 및 자동화된 배포 환경을 제공합니다.

2. 프런트엔드 (Frontend)
메인 프레임워크: React.js
선정 이유: AI 및 데이터 시각화 분야에서 대시보드 개발에 가장 널리 사용되는 프런트엔드 라이브러리입니다. 컴포넌트 기반 아키텍처로 UI 재사용성이 높고, 대규모 애플리케이션 개발에 적합합니다. Meta(구 Facebook)에서 개발하고 유지보수하며, 압도적인 커뮤니티와 생태계를 자랑합니다. 비교적 Breaking Change가 적고 하위 호환성 관리가 용이합니다.
상태 관리 라이브러리: React Query (TanStack Query) 또는 Redux Toolkit
선정 이유:
React Query: 서버 상태 관리 및 비동기 데이터 처리에 특화되어 백엔드 API와의 연동을 효율적으로 관리합니다. 캐싱, 데이터 동기화, 에러 처리 등을 자동화하여 개발 편의성을 높입니다.
Redux Toolkit: Redux의 복잡성을 줄이고 효율적인 상태 관리를 돕는 공식 권장 도구로, 대규모 애플리케이션에서 예측 가능한 상태 관리에 유용합니다.
데이터 시각화 라이브러리: Chart.js 또는 Recharts
선정 이유:
Chart.js: 경량의 유연한 JavaScript 차트 라이브러리로, 다양한 종류의 차트를 쉽게 구현할 수 있으며 뛰어난 시인성과 애니메이션 효과를 제공합니다.
Recharts: React 기반의 선언적인 차트 라이브러리로, React 컴포넌트와의 통합이 용이하며 깔끔하고 커스터마이징 가능한 차트를 제공합니다. 두 라이브러리 모두 활발하게 유지보수되며 커뮤니티 지원이 좋습니다.
UI 컴포넌트 라이브러리: MUI (Material-UI) 또는 Ant Design
선정 이유:
MUI: Google Material Design 기반의 React UI 컴포넌트 라이브러리로, 완성도 높은 디자인과 다양한 컴포넌트를 제공하여 개발 시간을 단축하고 일관된 UI/UX를 구현합니다.
Ant Design: 기업용 애플리케이션 개발에 최적화된 React UI 라이브러리로, 강력한 기능과 깔끔한 디자인을 제공하며 대시보드 구축에 필요한 다양한 컴포넌트를 포함합니다. 두 라이브러리 모두 대규모 기업에서 사용되며 활발히 유지보수됩니다.

[기술 스택 요약]
백엔드: Django, Django Rest Framework, PostgreSQL, Pandas
프런트엔드: React.js, React Query 또는 Redux Toolkit, Chart.js 또는 Recharts, MUI 또는 Ant Design
배포: Railway

[개발 방식 제안]
1. 엑셀 파일 업로드 및 데이터베이스 연동 방식 (권장)
관리자 업로드 기능: Django의 관리자 페이지 또는 별도로 구현된 커스텀 페이지를 통해 관리자가 이카운트에서 다운로드한 엑셀 파일을 직접 업로드합니다.
데이터 처리: 백엔드(Django)는 업로드된 파일을 Pandas 라이브러리를 사용하여 파싱하고, 유효성 검사 후 PostgreSQL 데이터베이스에 저장합니다.
시각화: 프런트엔드(React)는 저장된 데이터를 백엔드 API를 통해 조회하여 다양한 차트 라이브러리(Chart.js 또는 Recharts)로 시각화합니다.
장점: 보안성이 높고, 관리자가 데이터 유효성을 직접 확인하며 업로드할 수 있어 데이터 신뢰도를 높일 수 있습니다.
2. 지정된 파일 경로 자동 연동 방식 (협의 및 제안 가능)
파일 모니터링: 백엔드(Django)에서 Celery Beat 등의 스케줄러를 활용하여 특정 서버 경로에 저장되는 엑셀 파일을 주기적으로 모니터링합니다.
자동 파싱 및 저장: 새로운 파일이 감지되면 Pandas를 사용하여 자동으로 데이터를 파싱하고 데이터베이스에 저장합니다.
장점: 사용자의 수동 작업 없이 데이터 연동이 자동화되어 편리합니다.
고려사항: 이카운트에서 파일을 특정 서버 경로에 자동으로 저장하는 기능 지원 여부, 서버 파일 시스템 접근 및 보안 설정, 파일 동기화 이슈 등을 사전에 검토해야 합니다.
3. 로그인 기능
인증 시스템: Django Rest Framework JWT (JSON Web Token) 또는 OAuth2를 활용하여 안전한 토큰 기반 인증 시스템을 구축합니다.
사용자 관리: Django의 기본 사용자 모델을 활용하여 쉽게 구현 가능하며, 허가된 사용자만 대시보드에 접근할 수 있도록 권한을 관리합니다.
